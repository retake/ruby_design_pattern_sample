概要
* 複雑なオブジェクトを作り出すためのパターン。（正しいクラスを選択してオブジェクトを作るパターンは、factory_method）
* 成果物としてのオブジェクトと生成のためのオブジェクトを分離する。
* 生成用のオブジェクトで必要な情報を管理し、成果物としてのオブジェクトを作成する
* 無効なオブジェクト作成を防ぐ仕組みも生成用オブジェクトに入れられる（validationを作る）

サンプル
* オブジェクト作成対象のProductクラスには、MainPartsとSubPartsがあり、それぞれがどの様な値を取るか、作成時に決める必要がある。
* 通常の手順では、必要なオブジェクトを直接newしてproductに渡している。どのクラスをnewするのか、どの様な引数を渡すのか、という情報をクライアントオブジェクトが知っている必要が出てしまい、さらに攻勢が複雑化した場合（例えばMainPartsのクラス変数としてさらに下位のクラスのオブジェクトが必要になる）の対応が困難になって行く
* Builderを使った作成では、この後各パーツがどんなに複雑化して行っても、対応するメソッドを実行する形で構成を決める事が出来る。やる事べきこともこちらの方が明確である。
